1注册GitHub账号

如果已有GitHub账号，跳过此步

操作步骤

  1. 打开浏览器，访问 https://github.com
  2. 点击右上角的 "Sign up" 按钮
  3. 输入邮箱地址，设置密码，输入用户名
    建议用户名：chinasafetypro（如果你注册了这个用户名，后续可以用 chinasafetypro.github.io 直接访问网站）
  4. 完成验证题（拖动方块到正确位置）
  5. 去邮箱点击验证链接
  6. 回到GitHub，按提示设置偏好（全选默认即可）
GitHub在国内可以直接访问，不用翻墙。加载图片可能稍慢，不影响操作。

2创建仓库

仓库是用来存放网站文件的地方

操作步骤

  1. 登录GitHub后，点击右上角 + 图标 → "New repository"
  2. 填写表单：
    • Repository name（仓库名）：你的用户名.github.io
      例如：如果你的用户名是 chinasafetypro，就填 chinasafetypro.github.io
      这个格式很重要！只有仓库名是"用户名.github.io"才能用免费域名直接访问。
    • Description（描述）：填 China Safety Pro - 东南亚工厂安全合规专家
    • Public / Private：选 Public（公开），GitHub Pages免费版只支持公开仓库
    • Initialize：勾选 "Add a README file"
  3. 点击 "Create repository"（绿色按钮）
[仓库创建页面示意]
┌──────────────────────────────────────┐
│ Repository name: chinasafetypro.github.io │
│ Description: China Safety Pro │
│ ○ Public ● Private │
│ ☑ Add a README file │
│ │
│ [Create repository] (绿色按钮) │
└──────────────────────────────────────┘
如果你后续购买了自定义域名（如 chinasafetypro.com），仓库名不一定要用"用户名.github.io"格式，但用这个格式最简单，可以先上线再改。

3上传网站文件

共32个文件，约620KB，一次性上传

方法A：浏览器上传（推荐）

最简单，不用安装任何软件。适合第一次部署。

方法B：Git命令行推送

适合后续更新。需要安装Git并配置Personal Access Token。

方法A：浏览器上传（推荐新手）

  1. 进入你刚创建的仓库页面
  2. 点击 "Add file" → "Upload files"
  3. 打开你电脑上的文件夹：github-pages-deploy
    这个文件夹在 WorkBuddy 工作区目录下，路径类似：
    C:\Users\Administrator\WorkBuddy\2026-05-31-11-24-40\github-pages-deploy
  4. 选中文件夹里的所有32个文件（包括点开头的隐藏文件 .nojekyll），拖拽到上传区域
    需要上传的32个文件清单： .nojekyll ← 告诉GitHub不要用Jekyll处理 CNAME ← 自定义域名指向 robots.txt ← 搜索引擎爬虫指令 sitemap.xml ← 站点地图 index.html ← 网站首页 pricing.html ← 产品定价页 404.html ← 404错误页 privacy.html ← 隐私政策 terms.html ← 服务条款 东南亚工厂安全合规自查清单50条.html 东南亚工厂安全合规自查清单100条完整版.html 30天YouTube内容日历_完整版.html 30天YouTube内容日历_完整脚本版.html YouTube系列脚本_10期完整版.html 模块2_ISO45001核心条款速通_录制脚本+PPT大纲.html 视频录制生产指南.html 部署上线指南.html 程序文件模板_01 ~ 15（共15个.html文件）
  5. 在底部 Commit message 框里输入：首次上传网站文件
  6. 点击 "Commit changes"（绿色按钮）
  7. 等待几秒钟，文件上传完成
注意：浏览器上传有单次100个文件的限制，32个文件没问题。但如果后续要一次性上传很多文件，建议用方法B。

方法B：Git命令行推送

部署包文件夹（github-pages-deploy）里已经初始化了Git仓库并做好了首次提交。你只需要：

# 1. 先在GitHub上创建Personal Access Token # GitHub → Settings → Developer settings → Personal access tokens # → Generate new token → 勾选 repo → 生成并复制token # 2. 在github-pages-deploy文件夹中执行： git remote add origin https://github.com/你的用户名/chinasafetypro.github.io.git git push -u origin main # 会提示输入用户名和token（token就是密码）
方法B的好处是后续更新网站只需要改文件 → git add → git commit → git push，比浏览器拖拽方便得多。

4启用GitHub Pages

让GitHub知道"把这个仓库当网站来部署"

操作步骤

  1. 进入你的仓库页面（刚才上传文件的那个页面）
  2. 点击页面顶部的 "Settings"（设置标签）
  3. 在左侧菜单找到 "Pages"（在 "Code and automation" 下面）
  4. 找到 "Build and deployment" 区域
  5. 在 Source 下拉菜单中选择 "Deploy from a branch"
  6. 在 Branch 下拉菜单中选择 "main"，文件夹选择 "/ (root)"
  7. 点击 "Save"
[Pages设置页面示意]
┌──────────────────────────────────────┐
│ Build and deployment │
│ Source: [Deploy from a branch ▼] │
│ Branch: [main ▼] / [(root) ▼] │
│ │
│ [Save] │
└──────────────────────────────────────┘
几秒钟后，页面顶部会出现一个绿色提示框：
"Your site is published at https://你的用户名.github.io/"
点击这个链接，你的网站就上线了！
如果显示404或"页面不存在"，等3-5分钟再刷新。GitHub Pages部署需要一点时间生效。
网站会自动启用HTTPS（浏览器地址栏的🔒图标）。

5绑定自定义域名（可选）

如果你购买了 chinasafetypro.com，把它绑定到GitHub Pages
这一步需要你先购买域名。推荐渠道：
• Cloudflare（https://dash.cloudflare.com） — 推荐，自带CDN加速和DDoS防护
• Namecheap（https://www.namecheap.com） — 价格便宜，管理方便
• 阿里云万网（https://wanwang.aliyun.com） — 国内常用

第一步：在域名商配置DNS记录

到你的域名管理面板，添加以下4条DNS记录：

类型 名称 内容 代理
CNAMEwww你的用户名.github.ioDNS only
CNAME@你的用户名.github.ioDNS only
A@185.199.108.153DNS only
A@185.199.109.153DNS only
Cloudflare用户：代理状态选"DNS only"（灰色云朵），不要开橙色云朵代理，否则GitHub Pages的SSL会报错。

第二步：在GitHub仓库配置域名

  1. 进入仓库 → Settings → Pages
  2. 在 "Custom domain" 框中输入：chinasafetypro.com
  3. 勾选 "Enforce HTTPS"（等DNS生效后才能勾选，可能需要等几分钟）
  4. 点击 Save
DNS生效后（通常5-30分钟，最长48小时），你就可以通过 chinasafetypro.com 访问网站了。

6配置第三方服务

让支付、邮件、统计功能真正可用

6.1 Stripe支付（信用卡）

  1. 注册 Stripe 账号：https://dashboard.stripe.com/register
  2. 完成商家认证（需要营业执照或个人信息）
  3. 创建3个产品（对应3档定价）：
    • 自学版 — $69
    • 带教版 — $179
    • 企业版 — $697
  4. 获取 Publishable Key（在 Developers → API keys）
  5. 在 index.html 中搜索 TODO_STRIPE，替换 Publishable Key
最简方案：不用改代码。在Stripe后台创建产品后，用"Payment Links"功能生成支付链接，直接替换 index.html 中的按钮链接即可。

6.2 PayPal支付

  1. 注册 PayPal Business 账号：https://www.paypal.com/business
  2. 获取 Client ID（在 Applications → My Apps → 创建App）
  3. 在 index.html 中搜索 TODO_PAYPAL，替换 Client ID

6.3 Mailchimp邮件订阅

  1. 注册 Mailchimp：https://mailchimp.com
  2. 创建 Audience（受众列表）
  3. 进入 Audience → Signup forms → Embedded forms → 复制 form action URL
  4. 在 index.html 中搜索 TODO_MAILCHIMP，替换 action URL

6.4 Google Analytics 4

  1. 注册 GA4：https://analytics.google.com
  2. 创建 Property → 获取 Measurement ID（格式：G-XXXXXXXXXX）
  3. 在 index.html 中搜索 TODO_GA4，替换 Measurement ID

确认所有功能正常工作
  • 访问 https://你的用户名.github.io/，首页正常加载
  • 检查中英文切换按钮是否正常工作

?常见问题

Q：国内能访问GitHub Pages吗？

A：大部分时候可以，偶尔会不稳定。如果需要给国内客户看，建议绑定自定义域名并使用Cloudflare CDN加速。注意：自定义域名如果没备案，国内运营商可能偶尔拦截。

Q：GitHub Pages有流量限制吗？

A：免费版有100GB/月的带宽限制和10GB的存储限制。对你这个网站（32个HTML文件共约620KB）来说完全够用，除非你放了很大的视频文件。

Q：怎么更新网站？

A：最简单的方式：GitHub仓库页面 → Add file → Upload files → 上传修改后的文件 → Commit。
或者用Git命令行：改文件 → git add → git commit → git push。

Q：URL里有中文文件名会有问题吗？

A：不会。GitHub Pages支持UTF-8编码的URL，中文文件名会被自动编码为%XX格式。但搜索引擎URL中显示的是编码后的地址，不太美观。后续可以考虑改成英文名。

Q：需要备案吗？

A：GitHub Pages的服务器在美国，不需要ICP备案。但如果绑定自定义域名后想要在国内稳定访问，备案是更好的选择（需要国内服务器）。现阶段用Cloudflare做DNS解析即可。

Q：配置TODO在哪里找？

A：在 index.html 文件中搜索以下关键词即可定位需要替换的内容：
• TODO_STRIPE — Stripe Publishable Key
• TODO_PAYPAL — PayPal Client ID
• TODO_MAILCHIMP — Mailchimp form action URL
• TODO_GA4 — Google Analytics Measurement ID

Q：后续更新文件怎么办？

A：修改本地 github-pages-deploy 文件夹中的文件后：
git add . → git commit -m "更新说明" → git push origin main
GitHub Pages会在1-3分钟内自动更新网站。
